# Customs Verification — Field Scanner

This repository contains a React + TypeScript web application (Vite) designed as a field officer portal for customs product verification. It provides a mobile-first UI for capturing product images using the device camera, verifying product details (serial number, HSN code) against a Supabase-hosted database, and logging audit records.

This README documents the project's purpose, architecture, data model, environment variables, development workflow, and deployment notes.

## About

Purpose: provide a streamlined interface for customs officers to scan product labels, verify product authenticity and HS (Harmonized System) codes, and maintain a tamper-evident verification log (with images, location, timestamps).

Key features:
- Camera scanning with viewfinder overlay
- Manual entry fallback (serial number + HSN)
- Supabase authentication (email/password) and storage for captured images
- Verification logging (verification_logs table)
- Export verification history to CSV

## Architecture and key files

- Framework: Vite + React + TypeScript
- UI: Tailwind CSS + shadcn-style component collection (Radix + utility components)
- State / server cache: @tanstack/react-query
- Auth / DB / Storage: Supabase (Postgres + Storage)

Key files:
- `src/main.tsx` — app entry that mounts React
- `src/App.tsx` — top-level providers, router setup, and route mapping
- `src/pages/*` — main pages (Index, Auth, Scanner, ManualEntry, History, NotFound)
- `src/integrations/supabase/client.ts` — Supabase client instance (reads env vars)
- `src/integrations/supabase/types.ts` — autogenerated DB typings (products, user_profiles, verification_logs)
- `src/components/VerificationResult.tsx` — modal / UI for showing verification outcome
- `public/` — static assets (manifest, robots, etc.)

Package manifest: `package.json` (script shortcuts: `dev`, `build`, `preview`, `lint`)

## Pages / Routes

- `/` — Landing / index page (`src/pages/Index.tsx`) — checks session and routes to the scanner
- `/auth` — Authentication (signin/signup) (`src/pages/Auth.tsx`)
- `/scanner` — Camera scanning UI (`src/pages/Scanner.tsx`)
- `/manual-entry` — Manual verification form (`src/pages/ManualEntry.tsx`) — accepts optional captured image URL
- `/history` — Verification logs listing and CSV export (`src/pages/History.tsx`)

## Data model (Supabase)

The project includes generated TypeScript types in `src/integrations/supabase/types.ts`. Important tables (public schema):

- `products` — known products inventory
  - `id` (string)
  - `serial_number` (string)
  - `model_number` (string)
  - `hsn_code` (string)
  - `product_description`, `brand`, `declared_value`, `country_of_origin`, `image_url`, `barcode`, `created_at`

- `user_profiles` — optional profile enrichment for Supabase users
  - `id`, `full_name`, `badge_number`, `department`, `role`

- `verification_logs` — stores each verification attempt
  - `id`, `user_id`, `product_id` (nullable), `captured_image_url`, `extracted_serial`, `extracted_hsn`, `verification_status`, `mismatch_reason`, `latitude`, `longitude`, `ocr_raw_text`, `created_at`

Supabase storage: a bucket (example used in code) named `verification-images` is used to upload captured photos and retrieve a public URL.

## Environment variables

Create a `.env` file (or set environment variables in your host) with the following keys. Vite expects `VITE_` prefix to be exposed to the client.

- `VITE_SUPABASE_URL` — your Supabase project URL
- `VITE_SUPABASE_PUBLISHABLE_KEY` — Supabase anon/public key

These are used by `src/integrations/supabase/client.ts` to create the Supabase client.

Note: Keep your Supabase service role key out of frontend code. Only use the publishable (anon) key in the client.

## Local development

Prerequisites:
- Node.js (recommended: 18+)
- npm or pnpm

Install dependencies and run dev server:

```powershell
cd path\to\scan-verify-swift-main
npm install
npm run dev
```

By default, Vite serves the app at `http://localhost:5173`. You can open that on a mobile device (same network) for camera testing.

Auth flow:
- The app uses Supabase email/password auth. Sign up a test user via `/auth`.

Storage & CORS:
- The browser will upload captured images directly to Supabase Storage using the public key. Ensure the storage bucket `verification-images` exists and has appropriate public or signed-url access rules.

## Building & previewing

Production build:

```powershell
npm run build
npm run preview
```

This produces a production build in `dist/` (Vite default). The preview server serves the built assets locally.

## Testing & QA notes

- Camera permissions: the app requests camera access when starting the scanner. Test on devices with a back camera and ensure HTTPS (or localhost) to allow camera access.
- Geolocation: manual entry uses the browser geolocation API to record coordinates. If unsupported, it falls back to { latitude: 0, longitude: 0 } in the code.
- OCR: the current MVP simulates OCR by routing captured images to the manual entry page. Integrate an OCR service (Google Vision, Tesseract.js, AWS Rekognition) to automate extraction.

## Security & data considerations

- Do not commit Supabase service role keys or other secrets. Use environment variables in your deployment platform.
- Consider signed URLs for image uploads if you do not want public-read images.
- For production, enable RLS (Row Level Security) and define policies so users only read their own verification logs (or as appropriate for roles).

## Next steps / Improvements
